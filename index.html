❓ GUÍA DE USO

1 · CÓMO PUBLICAR UNA NUEVA SEMANA

  1. Asegúrate que los 11 archivos Excel del cierre estén actualizados en tu PC o SharePoint.
  2. Abre la página de carga: cargar.html.
  3. Completa el código de semana (ej. S37) y la fecha de corte (sábado del cierre).
  4. Arrastra los 11 Excel al dropzone (o click en "Seleccionar archivos").
  5. Click en "Procesar archivos →".
  6. Revisa los 6 KPIs del preview: Avance · CPI · SPI Meta · SPI Contractual · MO · Personal.
  7. Si todo luce bien, click en "🚀 Publicar al dashboard".
  8. Vuelve a este dashboard y espera 1-2 minutos. Apreta Ctrl+F5.
💡 Tiempo total: ~3 minutos. Los archivos se procesan en tu navegador — nada se sube hasta que confirmas "Publicar".

2 · CONFIGURAR EL TOKEN (SOLO LA PRIMERA VEZ)

Para que puedas publicar directamente sin entrar a GitHub, necesitas configurar un token una sola vez en tu navegador.

  1. Abre cargar.html. Verás una caja azul pidiendo el token.
  2. Genera el token en GitHub › Fine-grained tokens con permiso Contents: Read & Write sobre el repo esparq/e140-dashboard.
  3. Pégalo en el campo y click en "Guardar token".
  4. El estado cambia a ✓ Token OK (verde) en la parte superior.
⚠ Seguridad: el token se guarda solo en tu navegador. No lo compartas por canales públicos (grupos de WhatsApp, correo sin cifrado). Si sospechas que se filtró, revócalo en GitHub y genera uno nuevo.

3 · QUÉ HACER SI ALGO NO CARGA

  • Banner "Archivos faltantes": falta algún Excel al cargar. El dashboard usará los datos de la semana anterior para ese archivo. Puedes volver a publicar con el archivo completo después.
  • Celdas con "—": ese campo está vacío o no se pudo leer del Excel. No es un error, es un placeholder.
  • Error "Token inválido o expirado": el token de GitHub expiró. Genera uno nuevo y reemplázalo en ⚙ Config de cargar.html.
  • Semana no aparece en el dashboard: GitHub Pages tarda 1-2 minutos en publicar. Apreta Ctrl+F5 o espera.
  • Modo offline: sin internet, el dashboard muestra los últimos datos cacheados con un aviso.

5 · ACTUALIZACIÓN Y SINCRONIZACIÓN

El botón 🔄 Actualizar del header recarga los datos desde GitHub sin recargar la página entera. Úsalo cuando:

  • Acabas de publicar una semana nueva y quieres verla inmediatamente.
  • La sync está en rojo y quieres reintentar.
📡 Origen de datos: el dashboard lee prioritariamente los JSONs de la carpeta data/ del repo de GitHub. Si eso falla, cae automáticamente a datos demo.
